Understanding LCOM

LCOM = 1 - (SUM / (M * A))

Where:

  • SUM = Sum of methods accessing each property
  • M = Number of methods
  • A = Number of properties (attributes)
Low LCOM (0-0.4): Methods share properties, the class has a single responsibility.
High LCOM (0.6-1.0): Methods use different properties - consider splitting the class.

How to Improve Cohesion

Good Practices

  • Single Responsibility Principle - one class, one purpose
  • Group related properties and methods
  • Use composition over inheritance
  • Extract unrelated functionality to new classes

Warning Signs

  • lcom.warning_no_properties
  • lcom.warning_single_method
  • lcom.warning_large_classes
  • lcom.warning_god_classes
